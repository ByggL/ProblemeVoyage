Une Bibliothèque intéressante

Au début du développement du programme nous avons essayer d'utiliser une Bibliothèque nommée TSPLIB95.
C'est une Bibliothèque qui permet de lire les fichier TSP et d'associer une variable à chaque information.
Elle peut également résoudre directement le problème du voyageur de commerce.
Mais nous avons pensez que le projet ne devais pas être résolu de cette façon car il n'y aurais eu casiment aucun code venant de nous

La lecture du fichier et le stockage des variables

Nous avons donc commencer par lire le fichier en associant à une variable le nom , le Type , le commentaire et la dimension.
Ensuite il snous fallais une solution pour différencier les fichier avec une mattrice adjacente et ceux sans.
Nous avons remarquer la présence du text "NODE_COORD_SECTION" dans le fichier sans matrice d'adjacence , "NODE_COORD_SECTION" n'est pas présent dans le fichier avec la matrice.
Si le fichier contient une matrice adjacente celle-ci est stocker dans une liste sinon nous la calulons grace au coordonées fournis dans le fichier TSP.
Enfin nous stockons les coordonées dans une liste pour le calculs de la matrice d'adjacence.

La lecture du fichier nous a poser un petit car certain espace qui ont empécher la récupération direct des variables de nom, type, commentaire et dimension. Nous avons donc remplacer les " :" en ":"
